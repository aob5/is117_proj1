Tutorial 1: WebStorm Basics

This tutorial will explain step by step on how to setup WebStorm and connect with the Andrew File System (AFS)!

Step 1:

If you haven't already done so, download and install the WebStorm application to get started. Once installed, open it and click on "Create New Project".

Step 2:

Give a name to your new WebStorm project, and select the most recent version if possible. Then, click the blue "Create" button in the bottom right corner.

Step 3:

You will now see your project opened. To get started on logging into AFS, go to the "Tools" dropdown menu, and select the "Start SSH session..." option.

Step 4:

From here, you can simply login by providing a host name, your user credentials, and a port number. After providing that information, click on the blue "OK" button in the bottom right corner.

Step 5:

You will now see a sub-window showing your full access to the remote terminal from AFS. This terminal is typically used to control files and folders from AFS remotely.

Step 6:

Now, if you want to simulate the files and folders physically shown on AFS, you can do that by heading to the "Tools" dropdown menu, selecting the "Deployment" submenu, and then selecting the "Configuration..." option.

Step 7:

From here, you will want to click on the "+" symbol up at the top left corner, and select a connection protocol as a means to connect to the AFS network remotely.

Step 8:

Provide a name for the connection you are attempting to set up, then click on the blue "OK" button in the bottom right corner.

Step 9:

You will return to this window again, but now will be asked to provide information about the network you are attempting to connect to. Similarly to the SSH session login, provide a host name, port number, and user credentials. Also, provide any other links on the root path and web server url for any other configuration purposes. Now go ahead and click on the "Mappings" tab above.

Step 10:

From here, you will want to set the paths to which files and folders will be deployed onto and from AFS to the web. After everything is done being configured, click on the blue "OK" button in the bottom right corner.

Step 11:

You will now see on the right hand side a sub-window of the physical files and folders that appear on your AFS account. For better security, we would always want to make sure that when working on the local side, any files and folders used from the local storage gets sent automatically to the AFS side simultaneously. For that reason, head to the "Tools" dropdown menu for one last time, select the "Deployment" submenu, and then select the "Automatic Upload" option from there.

Step 12:

Now that "Automatic Upload" is set, initiate your first upload to AFS in order to start the automatic deployment. To do so, right-click whichever file you wish to upload to AFS, select the "Deployment" submenu option, then select the "Upload to AFS" option. You will now have successfully uploaded your files while making the automatic deployment active for your project.

Congradulations! You have now learned how to use WebStorm and set up the AFS!
