Tutorial 1: WebStorm Basics

This tutorial will explain step by step on how to setup WebStorm and connect with the Andrew File System (AFS)!

Step 1:

If you haven't already done so, download and install the WebStorm application to get started. Once installed, open it and click on "Create New Project".

Step 2:

Give a name to your new WebStorm project, and select the most recent version if possible. Then, click the blue "Create" button in the bottom right corner.

Step 3:

You will now see your project opened. To get started on logging into AFS, go to the "Tools" dropdown menu, and select the "Start SSH session..." option.

Step 4:

From here, you can simply login by providing a host name, your user credentials, and a port number. After providing that information, click on the blue "OK" button in the bottom right corner.

Step 5:

You will now see a sub-window showing your full access to the remote terminal from AFS. This terminal is typically used to control files and folders from AFS remotely.

Step 6:

Now, if you want to simulate the files and folders physically shown on AFS, you can do that by heading to the "Tools" dropdown menu, selecting the "Deployment" submenu, and then selecting the "Configuration..." option.

Step 7:

From here, you will want to click on the "+" symbol up at the top left corner, and select a connection protocol as a means to connect to the AFS network remotely.

Step 8:

Provide a name for the connection you are attempting to set up, then click on the blue "OK" button in the bottom right corner.

Step 9:

You will return to this window again, but now will be asked to provide information about the network you are attempting to connect to. Similarly to the SSH session login, provide a host name, port number, and user credentials. Also, provide any other links on the root path and web server url for any other configuration purposes. Now go ahead and click on the "Mappings" tab above.

Step 10:

From here, you will want to set the paths to which files and folders will be deployed onto and from AFS to the web. After everything is done being configured, click on the blue "OK" button in the bottom right corner.

Step 11:

You will now see on the right hand side a sub-window of the physical files and folders that appear on your AFS account. For better security, we would always want to make sure that when working on the local side, any files and folders used from the local storage gets sent automatically to the AFS side simultaneously. For that reason, head to the "Tools" dropdown menu for one last time, select the "Deployment" submenu, and then select the "Automatic Upload" option from there.

Step 12:

Now that "Automatic Upload" is set, initiate your first upload to AFS in order to start the automatic deployment. To do so, right-click whichever file you wish to upload to AFS, select the "Deployment" submenu option, then select the "Upload to AFS" option. You will now have successfully uploaded your files while making the automatic deployment active for your project.

Congradulations! You have now learned how to use WebStorm and set up the AFS!

Term: Definition:
Terminal

An emulator for working with a command-line shell from inside the IDE. Having a command line right inside the IDE lets you do things like running Git commands, and so on without switching windows and mental focus when you need to perform a quick command line task.
WebStorm

A powerful and intelligent IDE that gives you the best coding assistance for JavaScript, HTML and CSS and a wide range of modern web technologies. WebStorm is perfectly equipped for complex client-side development and server-side development with Node.js. It provides smart code insight, autocompletion, refactoring features, on-the-fly error prevention, and much more. Together with support for the popular frameworks such as AngularJS and Meteor and integrated instruments for testing, debugging and code analysis and integration with various VCS, WebStorm enhances your productivity and takes your development experience to a whole new level.
Integrated Development Environment (IDE)

A software application that provides comprehensive facilities to computer programmers for software development. An IDE normally consists of at least a source code editor, build automation tools and a debugger.
HTML5 Boilerplate

An HTML, CSS and JavaScript template (or boilerplate) for creating HTML5 websites with cross-browser capability.
Grunt

An ecosystem that WebStorm parses grunt files in to recognizing definitions of tasks and targets, shows tasks and targets in a tree view, lets you navigate between a task or a target in the tree and its definition in the grunt file, and supports running and debugging tasks and targets.
Linux

A Unix-like, open source and community-developed operating system for computers, servers, mainframes, mobile devices and embedded devices. Every version of the Linux operating system manages hardware resources, launches and handles applications, and provides some form of user interface.
HTTP

HTTP means HyperText Transfer Protocol. HTTP is the underlying protocol used by the World Wide Web and this protocol defines how messages are formatted and transmitted, and what actions Web servers and browsers should take in response to various commands.
Web Browser

A web browser, or simply "browser," is an application used to access and view websites. The primary function of a web browser is to render HTML, the code used to design or "mark up" webpages. Each time a browser loads a web page, it processes the HTML, which may include text, links, and references to images and other items, such as cascading style sheets and JavaScript functions. The browser processes these items, then renders them in the browser window.
HTML

Stands for "Hypertext Markup Language." HTML is the language used to create webpages. It is a standardized system for tagging text files to achieve font, color, graphic, and hyperlink effects on World Wide Web pages. "Hypertext" refers to the hyperlinks that an HTML page may contain. "Markup language" refers to the way tags are used to define the page layout and elements within the page.
CSS

Stands for "Cascading Style Sheet." Cascading style sheets are used to format the layout of Web pages. They can be used to define text styles, table sizes, and other aspects of Web pages that previously could only be defined in a page's HTML. It is used to define styles for your web pages, including the design, layout and variations in display for different devices and screen sizes.
Login

In the remote server configuration, a server runs on another computer (a remote host). To access files on the server, login the remote server using any of the FTP/SFTP/FTPS protocols.
File

In WebStorm, you can create files with language-specific extensions and some initial contents, thanks to file templates for most of the languages supported by the IDE. File templates are specifications of the default contents to be generated when creating a new file. Depending on the type of file you are creating, templates provide initial code and formatting that is expected to be in all files of that type (according to industry standards, your corporate policy, or for other reasons). WebStorm provides predefined templates for all supported file types, which are suggested when you are creating a new file.
Folder / Directory

In WebStorm, content is a collection of files with which you are currently working, possibly organized in a hierarchy of subfolders. The top-level folder of a project is its content root. You can add any folder as a part of your project. In the Settings/Preferences dialog, click Directories. The Directories page opens. To add a content root, click Add Content Root in the right-hand pane and then select the required folder in the dialog that opens.
AFS

The Andrew File System (AFS) is the primary academic computing environment at NJIT. AFS is a distributed file system comprised of multiple file and database servers and several hundred MacOS, Linux, Unix, and Windows client computers. A very wide spectrum of both open source and commercial applications, compilers, libraries, and utilities is available in AFS. NJIT calls our AFS file system Highlander AFS for unique branding purposes. Highlander AFS uses your UCID login and password. To best access Highlander AFS services, your computer should be an AFS client.
Rules for Naming Files and Folders on the Web

Filenames are part of your URL—and therefore an important part of your HTML. You can name your file nearly anything you choose, but you must follow a few rules of thumb to ensure that it displays correctly: don’t use special characters, don’t use spaces, start the file name with a letter, use all lowercase letters, keep your filenames short, and remember the file extension.
